# Управление пользователями (User Management App)

React-приложение для управления пользователями и группами в организации.

## Запуск проекта

### Требования
- Node.js (версия 14 или выше)
- npm (версия 6 или выше)

### Пошаговая инструкция по запуску
1. **Создайте новый React проект**
npx create-react-app user-management-app


2. **Перейдите в папку проекта**
cd user-management-app

3. **Установите дополнительные зависимости**
npm install react-router-dom react-icons

4. **Создайте папки для компонентов**
mkdir src/components
mkdir src/pages

5. **Замените содержимое файлов** в проекте на код из архивов (файлы приложены отдельно)

6. **Запустите приложение**
npm start

7. **Откройте браузер** и перейдите по адресу
http://localhost:3000

## Описание бизнес-логики проекта

### Что делает приложение?
Приложение позволяет управлять списком сотрудников компании: просматривать, добавлять, удалять, искать и сортировать пользователей, а также анализировать их распределение по группам.

### Основные сущности
- **Пользователь** - сотрудник с данными: имя, учетная запись, email, группа, телефон
- **Группа** - отдел или категория сотрудника (CEO, Managers, Financials и т.д.)
- **Unmanaged** - специальная группа для пользователей без назначенной группы

### Как это работает?
1. Данные загружаются из файла data.json при старте приложения
2. Пользователи отображаются в таблице на странице "Пользователи"
3. Группы формируются автоматически на основе данных о пользователях
4. Можно добавлять новых пользователей через форму
5. Можно удалять существующих пользователей
6. Поиск работает по всем полям таблицы
7. Сортировка работает по клику на заголовки столбцов

## Функциональность приложения

### Главная страница (/)
- Приветственный экран
- Три карточки с описанием разделов
- Список возможностей системы

### Страница пользователей (/users)
- Таблица со списком всех пользователей
- Поиск по всем полям (имя, email, группа, телефон)
- Сортировка по клику на заголовки столбцов
- Кнопка добавления нового пользователя
- Кнопка удаления пользователя
- Адаптивная верстка (горизонтальный скролл на телефонах)

### Страница групп (/groups)
- Статистика (всего пользователей, в группах, без группы)
- Прогресс-бары с распределением по группам
- Карточки групп со списком сотрудников
- Детальная таблица с анализом

## Технологии

## Выводы по проектированию UI

### Ручное проектирование (страница Users)

**Что делал(а) вручную:**
- Разрабатывала логику поиска и сортировки с useMemo
- Создавала компонент таблицы
- Добавляла форму добавления пользователя
- Настраивала адаптивность

**Плюсы ручного подхода:**
- Полный контроль над кодом
- Хорошая производительность
- Понимание каждого аспекта работы
- Легко вносить изменения

**Минусы ручного подхода:**
- Занимает много времени
- Нужно продумывать все детали самому

### Проектирование с помощью LLM (страница Groups)

**Что сгенерировала нейросеть:**
- Карточки групп
- Прогресс-бары для статистики
- Макет страницы групп

**Плюсы использования LLM:**
- Быстрое создание прототипа
- Креативные идеи для дизайна
- Экономия времени на рутине

**Минусы использования LLM:**
- Код требует проверки и доработки
- Нужно адаптировать под общий стиль
- Не всегда учитывается производительность
